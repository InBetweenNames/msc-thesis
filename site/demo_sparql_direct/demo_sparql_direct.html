Live Demo - New Semantics - SPARQL version - Direct Query Interface

A Demonstration of our new semantics remade using a SPARQL endpoint as the triplestore

This is a live demo of the Solarman application for Natural Language queries to event based semantic web data. Previously our demonstration used an in-program triplestore. To demonstrate the real-world applicability of our semantics, we have redesigned our demo to use a SPARQL endpoint instead as its triplestore. This improved version is faster and now operates directly on the semantic web.

Novel additions

In addition to these changes, we also demonstrate our support for chained prepositional phrases in our semantics here. We also demonstrate a novel way to handle the word 'by', as used in the passive form of a verb (e.g., 'discovered by'), by treating it directly as a preposition. In doing this, we can support queries such as 'which moon was discovered in 1877 by hall' without any added complexity to the semantics.

Query interface

In the input field below, you may type and run any valid query using the semantic combinators and functions.

Queries are written as expressions in the Haskell language.

Examples: which moons $ orbit jupiter

which planets $ orbited_ [by $ a (moon `that` discovered_ [by galileo])]

How to use

Unlike the natural language interface, this interface accepts expressions in the Haskell language to evaluate semantic queries. That is, you, the user, have the direct ability to construct and evaluate queries using our parser combinators. Try some of the examples above.

Some things to note: there are two transitive verbs used in Solarman, 'discover' and 'orbit'. Depending on your use of these verbs, the appropriate version must be called. "discover" is the active voice (i.e. "hall discovered a moon"), "discover'" is the active voice with support for prepositional phrases, and "discover_" is the passive voice with prepositional phrases. Note that we treat 'by' as a preposition in our grammar. An example query using the passive voice: 'phobos $ discovered_ [by hall]' (in English: "phobos was discovered by hall"). An example query in the active voice: 'hall $ discovered' phobos [in' 1877, with $ a telescope]' (in English "hall discovered phobos in 1877 with a telescope")

A full list of available functions is available here.

We permit the (.) and ($) operators from the prelude, and any function defined in that file, in queries. SafeHaskell prevents the evaluation of any other function.
